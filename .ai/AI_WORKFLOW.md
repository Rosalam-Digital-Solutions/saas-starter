# AI Workflow

## Before Coding

1. Read `.ai/PROJECT_CONTEXT.md`.
2. Read `.ai/CODING_RULES.md`.
3. Read the relevant domain docs for the task.
4. Inspect the files directly related to the change.
5. Identify risks and edge cases.
6. State assumptions before editing.

## While Coding

1. Make small changes.
2. Preserve existing style.
3. Avoid unrelated refactors.
4. Update docs if behavior or architecture changes.
5. Add tests when reasonable.

## After Coding

1. Run relevant commands.
2. Report files changed.
3. Report commands run.
4. Report failures honestly.
5. List any manual testing still required.
6. Update `.ai/TASKS.md` if task status changed.

## Final Response Template

- Summary
- Files changed
- Commands run
- Validation result
- Risks
- Manual testing
- Next steps
